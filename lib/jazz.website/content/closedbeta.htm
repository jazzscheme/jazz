JazzScheme 2.1.0

Which version should I use?

Binaries
The binaries are the easiest to install. They include the local .git repository needed to pull the latest changes and of course, already compiled binaries. Note that even if you install binaries, you can still pull and use the build system to build what changed since the binaries where compiled.

Sources
For sources, if you want to stay closely up-to-date on JazzScheme development or plan on contributing, you should choose the 'devel' version. It includes the local .git repository needed to pull the latest changes from the public repository or submit patches of contributions. Else, choose the non-'devel' version which is smaller.

Development or stable repository?

Dev
As the primary purpose of this closed-beta is for people to experiment, you should definitely use the "dev" repository. This should enable an easy turn around of bug reporting, fixing, pulling, ... Note that all the prepackaged releases below that include a Git repository point to the "dev" repository.

  • git://64.235.209.250/git/jazz_dev
  • http://www-etud.iro.umontreal.ca/~lasallej/git/jazz_dev (mirror updated every 15 mn)

Stable
Should you deliver a project using JazzScheme though, by all means choose the stable repository.
