Install binaries or build from sources?

Binaries
Installing binaries is the easiest method. They include the local .git repository needed to pull the latest changes and of course, already compiled binaries. Note that even when you install binaries, you can still pull from the repository to be up-to-date on the latest developments and the build system will only build what changed since the binaries where compiled.

Sources
For sources, if you want to stay closely up-to-date on JazzScheme development or plan on contributing, you should choose the 'devel' version. It includes the local .git repository needed to pull the latest changes from the public repository or submit patches of contributions. Else, choose the non-'devel' version which is smaller.

Development or stable repository?

Dev
As JazzScheme is still in Beta stage, using the "dev" repository is recommended. This should enable an easy turn around of bug reporting, fixing, pulling, ... Note that all releases that include a Git repository point to the "dev" repository.

  • git://github.com/jazzscheme/jazz_dev.git
  • git://64.235.209.250/git/jazz_dev (mirror)

Stable
Should you deliver a project using JazzScheme though, by all means choose the stable repository.

  • git://github.com/jazzscheme/jazz_stable.git
  • git://64.235.209.250/git/jazz_stable (mirror)
