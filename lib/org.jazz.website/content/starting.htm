Installation

  • Unzip the Jedi distribution anywhere you want. On Windows, most people put it in C:\JazzScheme but it will work in any location.

  • In the Bin folder, you can use either one of :

    Interpreter.exe
    This will launch Jedi in fully interpreted mode.
    Launcher.exe
    This will launch Jedi with key files compiled for execution speed. This is the recommended executable to use.

Operating Systems

The Jazz kernel runs on most operating systems. It is entirely written in mostly portable pure Scheme with the wonderfull Gambit Scheme system as primary target.

The multi-platform GUI is a high level framework written in Jazz that uses Cairo for backend. It runs on Windows and X11. The port to a native Mac interface will begin shortly.

Source Code

  • All source code in Jazz must exist inside a unit . There are 3 kinds of units:
    • Classes
    • Interfaces
    • Packages
  • To be able to reference a unit, it must be added to the workbench .
  • One very convenient place to tryout code is in your user class as all code executed in the console executes inside the user.

Workspaces

To manage the extensive number of UI features, Jedi uses workspaces to create virtual window spaces similar to Eclipse perspectives.

  • To switch workspace, use the buttons located at the right end side of the toolbar.
  • A Jazz window always belongs to one or more workspace. For example, a Jazz source code window can be viewed in both the Script and the Debugger workspaces.
